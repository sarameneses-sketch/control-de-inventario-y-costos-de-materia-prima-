
Requerimientos Funcionales (RF)
El sistema permitirá el registro y login de usuarios
	El sistema mostrará un catálogo detallado de servicios de marketing
El sistema debe procesar pagos en línea de forma segura
El sistema permitirá a los administradores gestionar el estado de los proyectos contratados

Requerimientos No Funcionales (RNF)
La aplicación debe cargar en menos de 3 segundos
La interfaz debe ser totalmente responsiva (móvil y escritorio)
Los datos de los clientes deben estar protegidos bajo normativas de privacidad de datos
